# Résumé — static asset

A self-contained, on-brand résumé for **Robert DeLanghe**, built on the Bounded
Systems design system rather than generated by a browser printing the live page.

| File | What |
| --- | --- |
| `resume.html` | The résumé — semantic HTML, one screen + print document |
| `resume.css` | Layout + the print stylesheet (colours come from the tokens) |

## Why build it instead of fixing print CSS on the live site

The previous PDF was the browser printing `robertdelanghe.dev/resume`, so it
carried the browser's own chrome — the date/time header (`6/23/26, 7:41 AM`),
the page URL, and a `Page 1 of 3` footer baked into every page. Those are
injected by the print dialog, not by the page, so no amount of site CSS removes
them cleanly.

This asset owns its own page box instead:

- **`@page { size: Letter; margin: 14mm 16mm }`** — real margins, not the
  browser default.
- `break-inside: avoid` on each experience entry and `break-after: avoid` on
  section labels, so an org never splits from its bullets across a page break.
- `print-color-adjust: exact`, so the forest header bar and accents survive the
  print pipeline.

## Print to a clean PDF

Open `resume.html` and print (⌘P / Ctrl-P):

1. **Destination:** Save as PDF
2. **Paper:** Letter
3. **Margins:** Default (the `@page` rule does the real work)
4. **Options → Headers and footers:** **off** ← this removes the date / URL /
   "Page X of Y" chrome the old export had
5. **Background graphics:** **on** ← keeps the forest header bar

## Design-system provenance

It links the brand sources directly — no copies, no drift:

```
../css/fonts.css     self-hosted Space Grotesk + IBM Plex Mono (no CDN)
../tokens/tokens.css design tokens — every colour is var(--bs-color-*)
../css/base.css      resets + element defaults
./resume.css         this surface's layout + print rules
```

Colours are never hand-picked here; they resolve from `tokens.json` → `tokens.css`.

## Preview locally

`fonts.css` and the stylesheets resolve over `file://`, so opening the file
works — but to match how it's served, run from the repo root:

```bash
python3 -m http.server 8080
# open http://localhost:8080/resume/resume.html
```

## Note on links

The header links (`GitHub`, `Writing`, `bounded.tools`, email) and the canonical
URL are inferred from the source PDF's text. Verify the `href`s in
`resume.html` against the live site before publishing.
